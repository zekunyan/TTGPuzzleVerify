TTGPuzzleVerify
Architecture

Native iOS core UIKit / SwiftUI / Objective-C Image + gradient puzzle backgrounds

A layered puzzle verification component: one shared native view, expressive configuration, predictable interaction state, and lightweight integration surfaces.

📱Integration Layer

Different app surfaces use the same verification engine and configuration model.

🧩SwiftUI

Wrap the native view with bindings for state, progress, result, and callbacks.

🎛️UIKit

Drop in a UIView, style it, control it with gestures or sliders, and react to verification.

🔗Objective-C

Use the generated Swift header with bridged enums, blocks, and delegate callbacks.

↓
🧠

TTGPuzzleVerifyView

The shared UIKit core owns rendering, puzzle geometry, dragging, verification state, and result reporting.

🎨Render

Background, blank cutout, puzzle piece, mask, and shadow composition.

✂️Pattern

Classic, square, circle, and custom UIBezierPath shape generation.

🕹️Interaction

Gesture and slider-driven x/y percentage movement with bounds.

✅Verify

Manual or automatic verification with tolerance, snap, failure, and reset states.

↓

✨Feature Modules

Compact config, broad visual and behavior coverage.

🖼️Backgrounds

Gradients, bundled images, or remote images.

🔷Shapes

Classic, square, circle, or custom path.

↻Behavior

Axes, reset, retry, manual verify, auto snap.

📊Metrics

Offsets, elapsed time, track points, interactions.

⚡Verification Flow

One simple runtime path for every demo.
1

Configure

Style, pattern, tolerance, axes, callbacks.

2

Render

Build the surface and movable piece.

3

Move

Drag or drive through slider percentages.

4

Verify

Success snap, failure feedback, reset state.

5

Report

Return result, state, and metrics.
